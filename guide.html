3. Local Development Setup

1

Clone the Repository

git clone https://github.com/yourusername/aurix.git
cd aurix
2

Install Dependencies

pnpm install

This installs all dependencies for both frontend and backend using pnpm workspaces.

3

Configure Environment Variables

# Frontend environment
cp .env.example apps/web/.env.local

# Backend environment
cp .env.example apps/server/.env

See Section 5 for detailed environment variable configuration.

4

Build Shared Packages

pnpm --filter @aurix/types build
pnpm --filter @aurix/utils build
5

Start Development Servers

# Start both frontend and backend
pnpm dev

# Or start individually:
pnpm --filter @aurix/web dev      # Frontend: http://localhost:3000
pnpm --filter @aurix/server dev   # Backend: http://localhost:3001
6

Verify Installation

# Test backend health
curl http://localhost:3001/health

# Expected response:
{
  "status": "ok",
  "timestamp": "2024-01-...",
  "services": {
    "redis": "connected",
    "firebase": "connected",
    "marketStream": "running"
  }
}

4. External Services Configuration

Firebase Setup

  1. Go to Firebase Console
  2. Create a new project (e.g., "aurix-trading")
  3. Enable Authentication (Email/Password provider)
  4. Create a Firestore database in native mode
  5. Go to Project Settings → Service Accounts
  6. Generate new private key (for backend)
  7. Note down the project ID and client email

Upstash Redis Setup

  1. Go to Upstash Console
  2. Create a new database
  3. Select the region closest to your users
  4. Copy the REST URL and token

Google AI (Gemini) Setup

  1. Go to Google AI Studio
  2. Create a new API key
  3. Copy the key for your environment variables

Groq Setup (Optional)

  1. Go to Groq Console
  2. Create an API key for AI fallback

5. Environment Variables

Frontend (.env.local)

# Firebase Configuration
NEXT_PUBLIC_FIREBASE_API_KEY=your_firebase_api_key
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=your_project_id
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=your_project.appspot.com
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=123456789
NEXT_PUBLIC_FIREBASE_APP_ID=1:123456789:web:abcdef

# Backend API
NEXT_PUBLIC_API_URL=http://localhost:3001
NEXT_PUBLIC_WS_URL=ws://localhost:3001/ws

Backend (.env)

# Server
PORT=3001
NODE_ENV=development
JWT_SECRET=your-super-secret-jwt-key-change-in-production

# Binance
BINANCE_WS_URL=wss://stream.binance.com:9443/ws

# AI APIs
GEMINI_API_KEY=your_gemini_api_key
GROQ_API_KEY=your_groq_api_key

# Upstash Redis
UPSTASH_REDIS_URL=https://your-redis-url.upstash.io
UPSTASH_REDIS_TOKEN=your_redis_token

# Firebase Admin
FIREBASE_PROJECT_ID=your_project_id
FIREBASE_CLIENT_EMAIL=firebase-adminsdk-xxxxx@your_project.iam.gserviceaccount.com
FIREBASE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n"

# Optional APIs
WHALE_ALERT_API_KEY=your_whale_alert_key
YAHOO_API_KEY=
⚠️ Important: Never commit your .env files to version control. They contain sensitive credentials.

6. Production Deployment

Option A: Vercel (Frontend) + Railway (Backend)

Deploy Frontend to Vercel

  1. Push code to GitHub
  2. Go to Vercel
  3. Import your repository
  4. Root Directory: apps/web
  5. Build Command: cd ../.. && pnpm install && pnpm --filter @aurix/web build
  6. Add environment variables from .env.local
  7. Update NEXT_PUBLIC_API_URL to your backend URL

Deploy Backend to Railway

  1. Go to Railway
  2. New Project → Deploy from GitHub repo
  3. Set root directory: apps/server
  4. Add all environment variables from .env
  5. Add healthcheck path: /health
  6. Generate domain or add custom domain

Option B: Render (Full Stack)

Web Service (Frontend)

Build Command: pnpm install && pnpm --filter @aurix/web build
Start Command: pnpm --filter @aurix/web start
Root Directory: apps/web

Web Service (Backend)

Build Command: pnpm install && pnpm --filter @aurix/server build
Start Command: pnpm --filter @aurix/server start
Root Directory: apps/server

Post-Deployment Checklist

  • Update CORS settings in backend with production URLs
  • Configure Firebase authorized domains
  • Verify Redis connection in production
  • Check WebSocket connectivity
  • Monitor error logs in both services

9. Troubleshooting

Common Issues

Build Errors

Error: Cannot find module '@aurix/types'

Solution: Build shared packages first

pnpm --filter @aurix/types build
pnpm --filter @aurix/utils build

Redis Connection Failed

Error: Redis connection refused

Solution: Check UPSTASH_REDIS_URL and token. Ensure using correct format.

Firebase Auth Errors

Error: Firebase App already initialized

Solution: This is a warning, not an error. The app handles multiple initializations.

CORS Errors

Error: CORS policy blocked request

Solution: Update CORS config in apps/server/src/index.ts with your domain

WebSocket Disconnections

Warning: WebSocket keeps reconnecting

Solution: Check firewall settings. Ensure WebSocket URL is correct.

Debug Mode

Enable debug logging:

# Backend
DEBUG=* pnpm --filter @aurix/server dev

# Frontend
NEXT_PUBLIC_DEBUG=true pnpm --filter @aurix/web dev

10. Security Best Practices

  • Never commit .env files to git
  • Use strong JWT secrets (32+ characters)
  • Enable Firebase App Check for production
  • Rate limit API endpoints (configured by default)
  • Validate all user inputs with Zod schemas
  • Use HTTPS in production
  • Regularly rotate API keys
  • Monitor Firebase Auth for suspicious activity
  • Set up Redis password/token rotation
🔒 Security Note: The application implements rate limiting, input validation, and proper CORS configuration by default. Always keep dependencies updated.
